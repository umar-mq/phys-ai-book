2.2 Physics, Properties & Collisions

Why Things Fall Downâ€‹

The Physics Engine Gazebo doesn't "know" physics. It uses a calculator called a Physics Engine. The default is usually Dart or Bullet. These engines calculate F=maF=maF=ma thousands of times per second.

Three Main Properties To make an object behave realistically, you must define:

  1. Mass: How heavy is it? (1kg vs 100kg).
  2. Friction: How rough is it? (Ice vs Sandpaper).
  3. Collision Shape: Where can it be touched?

The Collision Mesh Problemâ€‹

Beginners often ask: "Why did my robot fall through the floor?" Usually, it's because:

  • You forgot the <collision> tag in the URDF/SDF.
  • You spawned the robot inside the floor (Example: z=0.0). Physics engines hate this. They see two objects occupying the same space, get confused, and generate infinite force, launching the robot into space.
  • Fix: Always spawn your robot slightly above the ground (z=0.1).

Friction (Making it grip)â€‹

If your robot wheels spin but it doesn't move, you have no friction. In SDF (Simulation Description Format), we set <mu> (friction coefficient).

  • mu = 0.0: Ice.
  • mu = 1.0: Rubber on Concrete.
<surface>
  <friction>
    <ode>
      <mu>1.0</mu>
      <mu2>1.0</mu2>
    </ode>
  </friction>
</surface>
